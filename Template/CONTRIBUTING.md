# Workflow
This file contains templates for issues, commits and pull requests.

## Issues
### Issue template
**Issue Title**

**Description**   
A brief description of what this issue covers.

**Rationale**   
Why is this being done? Is it linked to a specific user story?

**Acceptance Criteria**   
- [ ] Criteria 1
- [ ] Criteria 2
- [ ] Criteria 3
- [ ] Criteria 4

You should also 
- Assign an appropriate label
- Add it to the project.
- Add it to the correct milestone and sprint.
- Assign 1 or more reviewers.

## Commits
### Commit codewords
add - A new feature or functionality.<br>
config - Changes to configuration files or project settings.<br>
docs - Documentation changes. <br>
fix - A bug fix. <br>
misc - Doesn't fit any of the other categories. <br>
revert - Reverting a previous commit. <br>
test - Adding or modifying tests. <br>

### Commit template
To use the provided template, write this command in your terminal

```bash
git config commit.template commit-template.txt
```

## Pull request (PR)
### Pull request template
**PR Title**   
Merging branch-name into branch-name

**Description**   
A brief explanation of what this PR does.

**Related Issue**   
Closes #issue-number

**Changes Made**   
- List the key changes, it should summarise the commits

**Checklist**
- [ ] I have tested this change locally (if applicable).
- [ ] I have updated the documentation (if needed).
- [ ] I have added/updated tests (if applicable).

You should also
- Assign an appropriate label.
- Add it to the project.
- Add it to the correct milestone and sprint.
- Assign 1 or more reviewers.
